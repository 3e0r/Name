# ?? 1. Что такое инфраструктурный пентест

Инфраструктурный пентест (или сетевой пентест) — это проверка безопасности всей «железной» и сетевой части компании:

* внутренняя сеть (AD, Windows-домены, сервера, рабочие станции)
* внешняя сеть (внешние IP, VPN, firewalls, почта)
* сетевые устройства (роутеры, свичи, WiFi)
* виртуальные среды (VMware, Proxmox, Hyper-V)
* облака (AWS, GCP, Azure)
* логические сегменты сети
* эксплуатация уязвимостей в сервисах и протоколах

Т.е. цель — проникнуть внутрь сети компании и получить максимум привилегий.

---

# ?? 2. Что такое Red Team (красная команда)

Red Team — это **не пентест**, а имитация работы реального хакера при определённой ТТП (tactics, techniques, procedures), обычно APT-подобной.

Главные отличия:

| Пентест                   | Red Team                                                                          |
| ------------------------- | --------------------------------------------------------------------------------- |
| ограниченный по времени   | длительная кампания (недели / месяцы)                                             |
| задача — найти уязвимости | задача — добиться бизнес-цели (кража данных, проникновение в сегмент, RCE на VIP) |
| «громкие» тесты можно     | нужно быть максимально скрытным                                                   |
| дают список дыр           | проверяют эффективность SOC, IR, SIEM                                             |
| можно шуметь сканерами    | нельзя шуметь (Evasion, OPSEC)                                                    |

Red Team ближе к реальному хакерству.

Red Team ? пентест.

### Этапы реальной RT-операции

1. **OSINT / разведка**

   * Сбор сотрудников, почт, телефонов
   * Выяснение используемых технологий (M365, VPN, Citrix, AD)
   * Сбор уязвимых сервисов

2. **Initial access (точка входа)**

   * Фишинг
   * MFA fatigue
   * Password spraying
   * Exploit публичного сервиса
   * Скомпрометированные cookie / refresh-токены
   * OAuth abuse

3. **Execution**

   * Запуск импланта (malware)
   * Обход EDR

4. **Persistence**

   * Scheduled tasks
   * WMI subscription
   * Azure OAuth tokens
   * Атакуем ADCS для долгосрочного присутствия

5. **Privilege escalation**

   * Local PrivEsc
   * Token manipulation
   * Kerberoast > S4U2Self > DA

6. **Lateral movement**

   * RDP, WinRM, SMB, DCOM, WMI
   * Proxying и pivoting (Chisel, Ligolo-NG, SOCKS трафик через имплант)

7. **Defense Evasion**

   * EDR bypass
   * Unhooking
   * Syscall-direct
   * Blocket ETW
   * AMSI bypass

8. **Impact (бизнес-цель)**

   * Доступ к почте гендиректора
   * Компрометация финансового сегмента
   * Захват домена
   * Кража секретов

## 2. Настройка лаборатории (AD-стенд) пошагово

Вот как поднять свой лабораторный стенд для Red Team-практики:

1. **Среда виртуализации**

   * Установи VirtualBox или VMware Workstation / Player.
   * Создай виртуальную сеть: `Host-only` (для внутренней сети) + NAT (для интернета, если нужно).

2. **Виртуальные машины**

   * **DC**: Windows Server (например, Server 2019/2022)
   * **Клиент**: Windows 10 / Windows 11
   * **Атакующая машина**: Linux (Ubuntu) + инструменты

3. **Настройка Active Directory**

   * На сервере: установить роль «Active Directory Domain Services» > продоминировать новый домен (например, `lab.local`).
   * Создать OU, группу “RedTeam”, добавить тестовых пользователей.
   * Настроить DNS внутри AD.

4. **Утилиты и инструменты**

   * Установить PowerShell (встроен)
   * Скачать BloodHound и SharpHound
   * Установить Mimikatz (только в учебной среде!)
   * Установить chisel / ligolo-ng для туннелей

5. **Безопасность**

   * Делай снапшоты ВМ после базовой конфигурации — чтобы был точкой возврата.
   * Не делай это на боевом окружении — только в учебном стенде.

## 1. Возможные SOC-специальности, на которые можно претендовать

Вот какие роли в SOC могут быть “входными” или промежуточными на твоем пути:

| Роль в SOC | Чем занимается | Почему это может быть полезно для Red Team / Infra Pentest |
|---|---|---|
| **SOC-аналитик / SOC-аналитик L1 (или просто “Аналитик SOC”)** | Мониторинг инцидентов, работа с SIEM, первичный анализ, triage тяжелых инцидентов | Изучаешь логи, атаки, учишься, как злоумышленник работает с точки зрения защиты > полезно для Red Team / Pentest |
| **SOC-аналитик L2 / L3 / Threat Hunter** | Более глубокий анализ, hunting, исследование инцидентов, написание правил в SIEM, корреляция событий | Понимаешь механику атак, тактики злоумышленников > переход в Red Team будет проще |
| **Инженер SOC / SOC Engineer** | Настройка SIEM и SOAR, интеграция источников логов, автоматизация, разработка корреляционных правил, возможно работа с response | Очень полезно: инфраструктурные знания + понимание, как устроен SOC, что важно, если ты хочешь моделировать атакующую сторону (Red) |
| **Консультант SOC / SOC Architect** | Проектирование и настройка SOC, процессы инцидент-реагирования, интеграция, политика | Меньше hands-on атаки, но хорошие знания архитектуры безопасности, которые пригодятся в Red Team / Pentest / Infra |

**Примеры вакансий SOC в Казани / России:**

- В **BI.ZONE**: «Аналитик Security Operations Center (SOC)» в Казани. ?cite?turn0search7?  
- **ICL Services**: SOC-аналитик L2 (в архивах вакансий). ?cite?turn0search8?  
- **Security Vision**: инженер-аналитик SOC — администрирование SIEM/SOAR, интеграция источников, правила корреляции. ?cite?turn0search6?  

Так что SOC — вполне реальный стартовый путь или параллельный путь, особенно если хочешь в итоге прийти в Red Team.

## 3. Как может проходить собеседование по каждой специальности и что спрашивать

Вот ориентиры, какие вопросы могут задавать, и что ты сам можешь спрашивать на собеседовании — в зависимости от роли.

### **A) SOC-аналитик (L1, L2, Threat Hunter)**

**Что могут спрашивать:**

- Что такое SIEM? Какие системы SIEM ты знаешь?  
- Как устроена цепочка логов: от источника (например, Windows) до SIEM?  
- Какие типы логов ты бы собирал / какую корреляцию правил написал?  
- Что такое IDS/IPS, как они работают?  
- Примеры инцидентов: если бы ты обнаружил подозрительный лог, что бы делал?  
- Основы сетевых протоколов: TCP, UDP, DNS?  
- Что такое MITRE ATT&CK? Какие техники ты знаешь?  
- Как ты расследовал бы потенциальный инцидент?  

**Что ты можешь спросить:**

- Какой стек SIEM/SOAR используется в компании?  
- Есть ли возможность участия в threat hunting?  
- Насколько часто происходит эскалация из L1 > L2 > IR?  
- Обучают ли, есть ли внутренние тренинги?  
- Можно ли со временем переходить на offensive задачи (пентест / Red Team)?  

**Как будет проходить:**

1. HR-этап: базовые вопросы, опыт, почему ИБ  
2. Техническое интервью: могут попросить разложить, как расследовать инцидент, задать ситуацию “симулируем атаку”  
3. Практика или тестовое задание: возможно кейс с логами / SIEM  
4. Финальное интервью (лид SOC или менеджер)  

---

### **B) SOC Engineer**

**Что могут спрашивать:**

- Знание SIEM и SOAR: как добавлять источники логов, создавать сценарии  
- Корреляция правил: как ты будешь ловить подозрительную активность  
- Автоматизация: скрипты, интеграции  
- Инфраструктура: сервер логи, сетевые устройства, облака  
- Как масштабировать SOC, как обеспечить надежность системы логов  
- Опыт работы с инцидент-реагированием: что произойдет, если правило сработало  

**Что ты можешь спросить:**

- Какая архитектура SOC: локальные сервера, облако?  
- Есть ли команды IR, Hunting, Red внутри компании?  
- Какие инструменты автоматизации используются?  
- Насколько “гибок” SOC: можно ли экспериментировать с новыми сценариями?  
- Есть ли возможность перейти в offensive / пентест?

**Как будет проходить:**

1. HR / скрининг  
2. Техническое интервью (может быть углублённое: инфраструктурные вопросы, автоматизация, логи)  
3. Тестовое задание: возможно настройка простого правила, korreляция  
4. Интервью с руководителем SOC или ИБ-отделом  

---

### **C) Infra Pentester / Red Team**

**Что могут спрашивать:**

- Опыт сетей: TCP/IP, DNS, маршрутизация  
- Active Directory: Kerberos, NTLM, делегации, pass-the-hash / pass-the-ticket  
- Примеры атак: как получить доступ из внешней сети, как переместиться латерально внутри сети  
- Техники post-exploitation: Mimikatz, Rubeus, BloodHound  
- Знание C2 / имплантов: писал ли ты что-то, на каком языке  
- EDR / AV: как обойти, какие техники знаешь  
- Опыт написания или использования “малвари”: загрузчики, стагеры, persistence  
- OPSEC: что значит, как минимизировать следы  

**Что ты можешь спросить:**

- Как устроена команда Red или пентест в компании  
- Какие инструменты / фреймворки используются (Cobalt Strike, Mythic и т.д.)  
- Как часто выполняются Red Team-операции  
- Есть ли сотрудничество с SOC / IR (Purple Team)  
- Какие требования к отчетности после операций  
- Можешь ли работать удаленно, гибкий график  

**Как будет проходить:**

1. HR-этап: “почему ты хочешь Red Team / Offensive”  
2. Техническое интервью: глубокие вопросы по сетям, AD, post-exploit, EDR  
3. Практическое задание: например, написать PoC, stager / loader, показать эксплойт, или разобрать кейс атаки  
4. Финальное интервью: технический лидер, возможно демонстрация портфолио или проектов
